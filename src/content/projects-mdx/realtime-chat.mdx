## Overview

A full-stack realtime chat application that supports instant messaging, presence, typing indicators, and secure socket authentication ‚Äî showcasing event-driven architecture and realtime system design.

---

## Problem

REST APIs are not enough for realtime chat because users expect:

- Instant message delivery
- Live online/offline presence
- Realtime typing indicators
- Conversations that update without refresh

---

## Solution

Built a websocket-powered chat system using **Socket.io** with a MERN backend.

**Highlights**
- Room-based messaging (isolated conversations)
- JWT-based socket authentication
- Persistent message storage in MongoDB
- Presence + typing events broadcast in realtime

---

## Key Features

### Messaging
- 1:1 realtime messaging
- Message timestamps
- Stored history (MongoDB)

### Presence & Typing
- Online/offline indicators
- Active user tracking
- Typing indicator events

### Security
- JWT login
- Socket middleware auth
- Protected rooms

---

## Architecture

**Frontend**
- Next.js + Tailwind UI
- State/context for active chat + messages

**Backend**
- Node.js + Express
- REST endpoints for user + chat history

**Realtime Layer**
- Socket.io events + rooms
- Broadcast to recipient + presence updates

**Database**
- MongoDB collections: Users, Conversations, Messages

---

## Messaging Workflow

1. User logs in  
2. Socket connects  
3. JWT verified in socket middleware  
4. User joins a conversation room  
5. Message emitted  
6. Saved in MongoDB  
7. Broadcast to recipient instantly  

---

## Challenges & Solutions

- **Socket Authentication:** Verified JWT in socket middleware to block unauthorized connections  
- **Room Isolation:** Generated unique room IDs per user pair to avoid message leaks  
- **Message Persistence:** Stored messages with timestamps for history loading  
- **Presence Tracking:** Maintained active socket sessions to track online users  

---

## Results

- Instant message delivery with event-driven flow
- Scalable room-based architecture
- Production-ready frontend + backend integration
